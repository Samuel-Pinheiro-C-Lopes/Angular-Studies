Data Binding with String Interpolation
With string interpolation, we can...

use double bracket notation to use a determined variable within our exported class to create dinamic data mutation.

this only happens while using forms module, from Angular/core, wich provides the ngModel directive.

As you can see here: {{ strings_interpolating[0] }}

Data Binding with Property Binding
With property binding, we can...

use the disabled property of a button and typeScript logic to enhace the website and have a dinamic button.

This will be deleted as well, using this and ngIf logic.

Data Binding versus String Interpolation

If you want to change a property, property binding is the way to chose. If you want to change the text displayed, string interpolation can do the trick!

Of course you could use property binding to do what String Interpolation does.

but, we can as well use (input), $event, and a function to do exactly what string interpolation does! =)
