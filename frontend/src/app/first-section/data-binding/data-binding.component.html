Data Binding with String Interpolation
String Interpolation

It's a way to embed a variable to a string, thus causing dynamic data mutation. The default notation is: double brackets with the variable name within - to then change the determined variable's inate value - therefor, being a fast and effective method to bind data throughout the application.

In the code sample bellow, a input HTML element has a ngModel directive - using forms module from Angular/core - whose value is defined once, and changes altogether with the input value.

Data Binding with Property Binding
Property Binding

Property binding works in a similar way to string interpolation, since it's defined as a method to mutate data in a dynamic way whenever it's needed to (but in that case, a property instead of exclusivaly a string) - as it's seem bellow, the button's "disabled" property changes when this input's value is missing (undefined - or in that case, specificaly a empty string). It's an effective way to stylize the page through the user's interaction.

This uses the disabled property of a button and typeScript logic to enhace the website's logic and dinamy - through Angular, of course.

This will be deleted as well, using this and ngIf logic - avalaible in the "directives" component.

Data Binding versus String Interpolation

If you want to change a property, property binding is the way to chose. If you want to change the text displayed, string interpolation can do the trick!

Of course you could use property binding to do what String Interpolation does...

but, we can - as well - use (input), $event, and a function to do exactly what string interpolation does! =)

A "paragraphChanger()" function receives - through $event as a parameter - the input element as argument, thus allowing it to change the text bellow, as the "innerText" property of the input bellow is binded to a "paragraphText2" variable in the backstages - it's value is mutated whenever the input value changes.
